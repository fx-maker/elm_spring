## 二、前端修改

### 1. AdminPage.vue（系统管理页面）
**文件路径：** `elmclient/src/views/AdminPage.vue`

**主要修改：**
- 用户列表新增"更改身份"下拉框
- 支持将普通用户升级为商家或管理员
- 支持将商家降级为普通用户

**核心代码：**
```html
<div class="card-row" v-if="u.userType !== 'ADMIN'">
  <span class="label">操作</span>
  <span class="value">
    <select class="type-select" @change="changeUserType(u.userId, $event)">
      <option value="">-- 更改身份 --</option>
      <option v-if="u.userType === 'USER'" value="BUSINESS">升级为商家</option>
      <option v-if="u.userType === 'BUSINESS'" value="USER">降级为用户</option>
      <option v-if="u.userType === 'USER'" value="ADMIN">升级为管理员</option>
    </select>
  </span>
</div>
```

**新增方法：**
```javascript
async changeUserType(userId, event) {
    const userType = event.target.value;
    if (!userType) return;
    if (!confirm(`确定要将该用户身份更改为"${this.typeName(userType)}"吗？`)) {
        event.target.value = "";
        return;
    }
    try {
        const res = await this.$axios.put(`UserController/updateUserType/${userId}/${userType}`);
        if (res.data.code === 200) {
            alert("修改成功");
            this.loadUsers();
        }
    } catch (e) {
        alert("修改失败");
    }
    event.target.value = "";
}
```

## 五、使用流程

### 管理员操作流程
1. 管理员登录系统
2. 进入"系统管理"页面
3. 在用户列表找到需要更改身份的用户
4. 选择下拉框中的目标身份
5. 确认更改

### 商家操作流程
1. 商家用户登录系统（userType = BUSINESS）
2. 进入"商家管理"页面
3. 可以：
   - 查看自己拥有的商家列表
   - 点击"新增商家"创建新商家
   - 点击商家卡片进入商家详情
   - 在商家详情中管理食品：
     - 新增食品
     - 编辑食品
     - 删除食品

## 六、注意事项

### MyBatis 多参数注解问题
在 MyBatis 中，如果方法有多个参数，必须使用 `@Param` 注解标注参数名，否则会导致 `500` 错误。

**错误写法：**
```java
public int updateOwner(Integer businessId, String ownerId);
```

**正确写法：**
```java
public int updateOwner(@Param("businessId") Integer businessId, @Param("ownerId") String ownerId);
```

### 服务重启
修改 Java 代码后需要重新编译并重启以下服务：
- user_server_10100 / user_server_10101
- business_server_10300 / business_server_10301
